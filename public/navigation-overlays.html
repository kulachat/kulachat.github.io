Navigation & Overlays

☰

Navigation

เอกสารนี้อธิบาย structure ของ navigation และ overlays แบบ Web Document ที่ใช้ร่วมกับ Design System Tokens กลาง เพื่อให้ทุกหน้ามี pattern เดียวกัน

Document shell

ใช้ topbar, tabbar, sidebar, content area และ right rail เป็นโครงเอกสารหลัก

Responsive behavior

desktop แสดง navigation ครบ ส่วน tablet/mobile ยุบ right rail และทำ sidebar เป็น block

Top navigation

Topbar เป็นพื้นที่สำหรับ brand, search, global actions และ account actions

ElementToken dependencyBehavior
Header height--doc-header-heightsticky top ระดับแรก
Search--ds-radius-pill, --ds-color-borderอยู่กลางบน desktop และซ่อนบน mobile
Actions--doc-accentใช้ accent สำหรับ primary action

Tab navigation

Tabbar ทำหน้าที่เชื่อมหน้าเอกสารหลัก และใช้ aria-current="page" เพื่อบอกหน้าปัจจุบัน

  • Active tab ใช้เส้นล่างสี accent และข้อความสี accent
  • Overflow บน mobile ให้ scroll แนวนอนแทนการบีบข้อความ
  • Static links ใช้ relative URL เพื่อเปิดได้ทั้ง local และหลัง deploy

Sidebar

Sidebar ใช้เป็น local navigation สำหรับหัวข้อในหน้าเดียวกัน แบ่งเป็นหมวดและ link ย่อย

Section title

ใช้ uppercase label เพื่อแบ่งหมวด เช่น Navigation, Related Pages

Current item

ใช้ border-left สี accent และ aria-current

Sticky position

desktop sticky ใต้ header+tabs, mobile กลายเป็น block ปกติ

Mobile menu

บน mobile โครงเอกสารควรลดความซับซ้อน โดยซ่อน search/right rail และให้ sidebar อยู่บนเนื้อหา

Mobile structure

  • Topbar เหลือ brand เป็นหลัก
  • Tabs เลื่อนแนวนอนได้
  • Sidebar เปลี่ยนจาก sticky rail เป็น card-like block

Overlays

Overlay controls เช่น Back to Top หรือ mobile drawer ควรวางเป็น utility ไม่ปนกับ footer

Back to Top

Mobile drawer

แสดง navigation ซ้อนบนเนื้อหาเมื่อ viewport แคบ

Command menu

ใช้สำหรับ search หรือ quick switcher เมื่อระบบต้องการ

Implementation pattern

หน้า navigation ใช้ layout กลางจาก web-document.css และเรียก token จาก design-system-tokens.css เพื่อให้ theme update ครั้งเดียวแล้วทุกหน้าเปลี่ยนตาม
