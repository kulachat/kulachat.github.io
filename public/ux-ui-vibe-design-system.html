Overview

โครงเอกสารนี้ใช้สำหรับอธิบาย Design System Token แบบเรียบง่ายเหมือน Web Document โดยยังคง design-system-tokens.css เป็น source of truth สำหรับทุกหน้า preview ในระบบ

Quickstart

เรียก token กลางจาก design-system-tokens.css แล้วให้ page-level CSS consume ผ่าน --ds-*

Theme update

แก้สี, spacing, radius, typography และ component tokens ที่ไฟล์เดียว แล้วหน้าอื่นจะตามไปด้วย

Structure layers

ระบบนี้แบ่ง token เป็น 4 ชั้น เพื่อให้ designer, developer และ Vibe Code workflow อ่านภาษาเดียวกัน

  • Primitive tokens ค่าดิบ เช่น palette, spacing, radius, typography และ shadow
  • Semantic tokens ค่าที่มีความหมาย เช่น background, surface, primary, border, text-muted
  • Component tokens ค่าที่ผูกกับ UI contract เช่น button height, field height, card padding
  • Pattern structure layout และ component ที่ใช้ data-component, data-state, data-variant

Token architecture

ตารางนี้คือ naming contract สำหรับนำไปใช้ต่อใน CSS variables, Tailwind theme, Figma variables หรือ React props

Layer Naming pattern Example
Primitive --ds-palette-{color}-{step} --ds-palette-teal-500
Semantic --ds-color-{role} --ds-color-primary-accent, --ds-color-surface
Component --ds-{component}-{property} --ds-button-height-md, --ds-card-padding
State [data-state="{state}"] active, selected, disabled, loading

Component contract

component ทุกตัวควรมีชื่อและ state ที่อ่านจาก DOM ได้ เพื่อให้การแก้ UI ไม่ต้องเดาจาก class เพียงอย่างเดียว

Button

data-component="button"

ออกแบบเลย!

Input field

data-component="input-field"

Choice chip

data-state="selected"

Selected Default

States and variants

state และ variant ควรใช้ attribute เป็น contract หลัก เช่น data-state, data-variant, data-size

  • Variant ใช้แยก intent เช่น primary, secondary, ghost, danger
  • State ใช้บอกสถานะ เช่น default, hover, active, selected, disabled, loading
  • Size ใช้กำหนด density เช่น sm, md, lg โดยอิง component token

Handoff standard

ถ้าจะ update token หรือ update theme ให้แก้ที่ public/design-system-tokens.css ก่อนเสมอ ส่วนไฟล์ HTML ควรทำหน้าที่เป็นเอกสารและตัวอย่าง implementation ที่เรียกใช้ token กลางเท่านั้น
